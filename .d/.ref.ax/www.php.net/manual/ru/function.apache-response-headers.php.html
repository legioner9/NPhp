    apache_response_headers

    (PHP 4 >= 4.3.0, PHP 5, PHP 7, PHP 8)

    apache_response_headers — Возвращает список всех HTTP-заголовков ответа Apache

    Описание

    apache_response_headers(): array|false

    Возвращает список всех HTTP-заголовков. Работает на веб-серверах Apache и FastCGI.

    Список параметров

    У этой функции нет параметров.

    Возвращаемые значения

    Массив HTTP-заголовков ответа Apache в случае успешного выполнения или false в случае возникновения ошибки.

    Примеры

    Пример #1 Пример использования apache_response_headers()

    <?php
    print_r    (apache_response_headers());
    ?>

    Вывод приведённого примера будет похож на: 

    Array
(
    [Accept-Ranges] => bytes
    [X-Powered-By] => PHP/4.3.8
)

    Смотрите также

    • apache_request_headers() - Получает список всех заголовков HTTP-запроса
    • headers_sent() - Проверяет, были ли отправлены заголовки
    • headers_list() - Возвращает список переданных заголовков (или готовых к отправке)

    <?php

    if (!function_exists('apache_response_headers')) {
    function     apache_response_headers () {
    $arh = array();
    $headers = headers_list();
    foreach (    $headers as $header) {
    $header = explode(":", $header);
    $arh[array_shift($header)] = trim(implode(":", $header));
    }
    return     $arh;
    }
    }

    ?>

    I can confirm athlet's experience with apache_response_headers() using PHP 5.1.6. When the resulting array is empty or only contains "X-Powered-By" instead of the full list of values, you'll need to switch off output_buffering _before_ the script has been executed, for example with the following directive in .htaccess:

    php_value "output_buffering" "0"

    it _doesn't_ work when you do a you ini_set('output_buffering', 0); in your script. Not sure why that is so - it is not a bug in PHP, but probably has to do with how Apache populates the request values.
