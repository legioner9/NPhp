    ftp_rename

    (PHP 4, PHP 5, PHP 7, PHP 8)

    ftp_rename — Переименовывает файл или директорию на FTP-сервере

    Описание

    ftp_rename(FTP\Connection $ftp, string $from, string $to): bool

    ftp_rename() переименовывает файл или директорию на FTP-сервере.

    Список параметров

    ftp

    An FTP\Connection instance.

    from

    Старое имя файла/директории.

    to

    Новое имя.

    Возвращаемые значения

    Возвращает true в случае успешного выполнения или false в случае возникновения ошибки.. В случае возникновения ошибки (например при попытке переименовать несуществующий файл) будет вызвана ошибка уровня E_WARNING.

    Список изменений

    Версия Описание
    8.1.0 Параметр ftp теперь ожидает экземпляр FTP\Connection; ранее ожидался ресурс (resource).

    Примеры

    Пример #1 Пример использования ftp_rename()

    <?php
    $old_file     = 'somefile.txt.bak';
    $new_file = 'somefile.txt';

    // установка соединения
    $ftp = ftp_connect($ftp_server);

    // проверка имени пользователя и пароля
    $login_result = ftp_login($ftp, $ftp_user_name, $ftp_user_pass);

    // попытка переименовать $olf_file в $new_file
    if (ftp_rename($ftp, $old_file, $new_file)) {
    echo     "Файл $old_file переименован в $new_file\n";
    } else {
    echo     "Не удалось переименовать $old_file в $new_file\n";
    }

    // закрытие соединения
    ftp_close($ftp);
    ?>

    This function isn't only able to rename files, but also folders. And it is not only able to rename them, but also move them, and, in the case of folders, their contents as well (so folders don't have to be empty to move). For example:

    <?php
    ftp_rename    ($conn_id, "./dir1/dir2/", "./dir3/");
    ?>

    Now the folder dir2 (which prevously was in folder dir1) has moved to the same folder as dir1, and it has kept its original contents (the content just moved along).
