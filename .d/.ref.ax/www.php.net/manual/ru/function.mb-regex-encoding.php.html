    mb_regex_encoding

    (PHP 4 >= 4.2.0, PHP 5, PHP 7, PHP 8)

    mb_regex_encoding — Устанавливает/получает кодировку символов для многобайтового регулярного выражения

    Описание

    mb_regex_encoding(?string $encoding = null): string|bool

    Устанавливает/получает кодировку символов для многобайтового регулярного выражения.

    Список параметров

    encoding

    Параметр encoding — это кодировка символов. Если он опущен или равен null, для него будет установлена внутренняя кодировка символов.

    Возвращаемые значения

    Если задан параметр encoding, то Возвращает true в случае успешного выполнения или false в случае возникновения ошибки. Если кодировка задана, то внутренняя кодировка символов не изменяется. Если параметр encoding не указан, то возвращается имя текущей кодировки символов для многобайтовых регулярных выражений.

    Список изменений

    Версия Описание
    8.0.0 Теперь параметр encoding может принимать значение null.

    Смотрите также

    • mb_internal_encoding() - Устанавливает/получает внутреннюю кодировку скрипта
    • mb_ereg() - Находит совпадение регулярному выражению с поддержкой многобайтовых кодировок

    mb_ereg functionality is provided via Oniguruma RegEx library and not via PCRE. mb_regex_encoding() does only support a subset of encoding names, compared to mb_list_encodings() and mb_encoding_aliases().
    Currently the following names are supported (case-insensitive):

    UCS-4LE

    CN-BIG5
    BIG-FIVE
    BIGFIVE
    EUC-KR
    EUC_KR
    eucKR
    KOI8-R
    KOI8R

    The list is a mixture of base names and aliases and applies to PHP 5.4.45 (Oniguruma lib v4.7.1), PHP 5.6.31 (v5.9.5), PHP 7.0.22 (v5.9.6) and PHP 7.1.8 (v5.9.6). Be aware of the inconsistency: mb_regex_encoding() accepts for example the base name 'UTF-8' and its only alias 'utf8', but it does not accept aliases 'utf16', 'utf32' or 'latin1'.

    Additionally note, that the informal name/alias 'latin9' for ISO/IEC 8859-15:1999 (including the Euro sign on 0xA4) is also not known by mb_list_encodings(). It can only be adressed as 'ISO-8859-15' or 'ISO_8859-15' and for mb_regex_encoding() solely as 'ISO-8859-15'.
