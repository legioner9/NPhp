    oci_field_precision

    (PHP 5, PHP 7, PHP 8, PECL OCI8 >= 1.1.0)

    oci_field_precision — Возвращает точность поля

    Описание

    oci_field_precision(resource $statement, string|int $column): int|false

    Возвращает точность поля column.

    Для полей типа FLOAT точность больше нуля, а масштаб равен -127. Если точность поля равна нулю, то тип поля - NUMBER. Иначе же, тип поля может быть описан как NUMBER(precision, scale).

    Список параметров

    statement

    Идентификатор допустимого OCI запроса.

    column

    Может быть номером поля (нумерация начинается с 1) или именем.

    Возвращаемые значения

    Возвращает точность в виде числа или false в случае возникновения ошибки.

    Примеры

    Пример #1 Пример использования oci_field_precision()

    <?php

    // Создайте таблицу:
    // CREATE TABLE mytab (c1 NUMBER, c2 FLOAT, c3 NUMBER(4), c4 NUMBER(5,3));

    $conn = oci_connect("hr", "hrpwd", "localhost/XE");
    if (!    $conn) {
    $m = oci_error();
    trigger_error(htmlentities($m['message']), E_USER_ERROR);
    }

    $stid = oci_parse($conn, "SELECT * FROM mytab");
    oci_execute($stid, OCI_DESCRIBE_ONLY); // используйте OCI_DESCRIBE_ONLY, если не получаете данных

    $ncols = oci_num_fields($stid);
    for (    $i = 1; $i <= $ncols; $i++) {
    echo     oci_field_name($stid, $i) . " "
    . oci_field_precision($stid, $i) . " "
    . oci_field_scale($stid, $i) . "<br>\n";
    }

    // Выведет:
    // C1 0 -127
    // C2 126 -127
    // C3 4 0
    // C4 5 3

    oci_free_statement($stid);
    oci_close($conn);

    ?>
