    odbc_pconnect

    (PHP 4, PHP 5, PHP 7, PHP 8)

    odbc_pconnect — Открывает постоянное соединение с базой данных

    Описание

    odbc_pconnect(
        string $dsn,
        string $user,
        string $password,
        int $cursor_option = SQL_CUR_USE_DRIVER
    ): resource|false

    Открывает постоянное соединение с базой данных.

    Функция очень похожа на odbc_connect(), за исключением того, что соединение на самом деле не закрывается после завершения работы скрипта. Последующие запросы на соединение с той же комбинацией dsn, user и password (с помощью odbc_connect() и odbc_pconnect()) могут повторно использовать постоянное соединение.

    Список параметров

    Для более подробной информации стоит обратиться к описанию odbc_connect().

    Возвращаемые значения

    Возвращает соединение ODBC или false в случае возникновения ошибки.

    Примечания

    Замечание: Постоянные соединения не действуют, если PHP используется как CGI.

    The following constants are defined for cursortype:
    - SQL_CUR_USE_IF_NEEDED
    - SQL_CUR_USE_ODBC
    - SQL_CUR_USE_DRIVER
    - SQL_CUR_DEFAULT

    With some ODBC drivers, executing a complex stored procedure may fail with an error similar to: "Cannot open a cursor on a stored procedure that has anything other than a single select statement in it". Using SQL_CUR_USE_ODBC may avoid that error. Also, some drivers don't support the optional row_number parameter in odbc_fetch_row(). SQL_CUR_USE_ODBC might help in that case, too.
