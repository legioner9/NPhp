    Установка

    --with-adabas[=DIR]

    Включить поддержку Adabas D. DIR - это каталог установки Adabas, по умолчанию /usr/local.

    --with-sapdb[=DIR]

    Включить поддержку SAP DB. DIR - каталог установки SAP DB, по умолчанию /usr/local.

    --with-solid[=DIR]

    Включить поддержку Solid. DIR - каталог установки Solid base, по умолчанию /usr/local/solid.

    --with-ibm-db2[=DIR]

    Включить поддержку IBM DB2. DIR - это каталог установки DB2, по умолчанию /home/db2inst1/sqllib.

    --with-empress[=DIR]

    Включить поддержку Empress. DIR - это каталог установки Empress, по умолчанию $EMPRESSPATH. Опция поддерживается только Empress версии 8.60 и выше.

    --with-empress-bcs[=DIR]

    Включить поддержку "Empress Local Access". DIR - это каталог установки Empress, по умолчанию $EMPRESSPATH. Опция поддерживается только Empress версии 8.60 и выше.

    --with-birdstep[=DIR]

    Включить поддержку Birdstep. DIR - это каталог установки Birdstep, по умолчанию /usr/local/birdstep.

    --with-custom-odbc[=DIR]

    Включить определённую пользователем поддержку ODBC. DIR - это каталог установки ODBC, по умолчанию /usr/local. Убедитесь, что вы определили CUSTOM_ODBC_LIBS и у вас есть несколько odbc.h в ваших включаемых каталогах. Например, вы должны определить следующее для Sybase SQL Anywhere 5.5.00 в QNX перед запуском скрипта настройки: 

       CPPFLAGS="-DODBC_QNX -DSQLANY_BUG"
   LDFLAGS=-lunix
   CUSTOM_ODBC_LIBS="-ldblib -lodbc".

    --with-iodbc[=DIR]

    Включить поддержку iODBC. DIR - это каталог установки iODBC, по умолчанию /usr/local.

    --with-esoob[=DIR]

    Включить поддержку Easysoft OOB. DIR - это каталог установки Easysoft OOB, по умолчанию /usr/local/easysoft/oob/client.

    --with-unixODBC[=DIR]

    Включить поддержку unixODBC. DIR - это каталог установки unixODBC, по умолчанию /usr/local.

    --with-openlink[=DIR]

    Включить поддержку OpenLink ODBC. DIR - это каталог установки OpenLink, по умолчанию /usr/local. Это то же самое, что и iODBC.

    --with-dbmaker[=DIR]

    Включить поддержку DBMaker. DIR - это каталог установки DBMaker, по умолчанию это место, где установлена последняя версия DBMaker (например, /home/dbmaker/3.6).

    Пользователи Windows должны включить php_odbc.dll, чтобы использовать этот модуль.

    To avoid «[Microsoft][ODBC Driver Manager] Data source name not found and no default driver specified»:

    - Find the ODBC manager in Windows control panel and check the "Drivers" tab. Please note that 64 bit Windows has two different managers for 32 [1] and 64 bit drivers: drivers must match your PHP installation.

    - If the required driver doesn't show up there, you need to install it. The free "Microsoft Access Database Engine 2010 Redistributable" package [2] is one of the possibilities for Access and Excel.

    - When connecting with odbc_connect(), make sure you are using the exact driver name in the DRIVER parameter: 'Microsoft Access Driver (*.mdb)' will not work if your driver is called 'Microsoft Access Driver (*.mdb, *.accdb)'.

    [1] C:\Windows\SysWOW64\odbcad32.exe
    [2] http://www.microsoft.com/en-us/download/details.aspx?id=13255
