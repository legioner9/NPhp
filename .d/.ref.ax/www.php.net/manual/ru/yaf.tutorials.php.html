    Примеры

    Пример #1 Классический пример каталога для приложения

    - index.php
- .htaccess
+ conf
  |- application.ini //Файл настроек
- application/
  - Bootstrap.php
  + controllers
     - Index.php //Контроллер по умолчанию
  + views
     |+ index
        - index.phtml //Шаблон вывода для действия по умолчанию
  + modules
  - library
  - models
  - plugins

    Пример #2 Введение

    index.php единственная точка входа в приложение, все запросы вы должны направлять через него (например, при помощи .htaccess в Apache + php_mod)

    <?php
    define    ("APPLICATION_PATH", dirname(__FILE__));

    $app = new Yaf_Application(APPLICATION_PATH . "/conf/application.ini");
    $app->bootstrap() //Выполнение методов, опрделенных в Bootstrap.php
    ->run();
    ?>

    Пример #3 Правила перенаправления

    #для apache (.htaccess)
RewriteEngine On
RewriteCond %{REQUEST_FILENAME} !-f
RewriteRule .* index.php

#для nginx
server {
  listen ****;
  server_name  domain.com;
  root   document_root;
  index  index.php index.html index.htm;

  if (!-e $request_filename) {
    rewrite ^/(.*)  /index.php$1 last;
  }
}

#для lighttpd
$HTTP["host"] =~ "(www.)?domain.com$" {
  url.rewrite = (
     "^/(.+)/?$"  => "/index.php/$1",
  )
}

    Пример #4 Конфигурация приложения

    [yaf]
;APPLICATION_PATH должна быть определена в index.php
application.directory=APPLICATION_PATH "/application/"

;product секция должна наследовать yaf
[product:yaf]
foo=bar

    Пример #5 Контроллер по умолчанию

    <?php
    class IndexController extends Yaf_Controller_Abstract {
    /* действие по умолчанию */
    public function indexAction() {
    $this->_view->word = "hello world";
    //или
    // $this->getView()->word = "hello world";
    }
    }
    ?>

    Пример #6 Шаблон вывода по умолчанию

    <html>
    <head>
    <title>Hello World</title>
    </head>
    <body>
    <?php echo $word;?>
     </body>
    </html>

 </head>

   hello world
 </body>
</html>

    When you use nginx & php-fpm config for yaf frame, you must set PATH_INFO support in nginx. Otherwise, it will not work.
    Demo :
    location / {
    try_files $uri $uri/ @path_rw;
    }

    #path_info is needed by some php frameworks like yaf/thinkphp
    location @path_rw {
    rewrite ^/(.*)$ /index.php/$1 last;
    }

    location ~ \.php {
    fastcgi_pass unix:/dev/shm/php-cgi.sock;
    fastcgi_index index.php;
    include fastcgi.conf;

    fastcgi_split_path_info ^(.+?\.php)(/.+)$;
    fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
    fastcgi_param SCRIPT_NAME $fastcgi_script_name;
    fastcgi_param PATH_INFO $fastcgi_path_info;
    }
